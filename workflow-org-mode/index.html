Workflow: Org mode and Orgzly

More than org-mode, I use Orgzly, the android app for org-mode. It can save the data in org files which are plain-text. These org files are synced to my computer using syncthing and processed in Emacs. Only the following files are managed this way.

  • Inbox: It holds 2 kinds of data
    • Thoughts that pop into my head which I want to follow up later
    • Usually when I read an article on web, I highlight interesting parts with hypothesis. However since I am using firefox which does not have an official hypothesis addon, some articles won't open in hypothesis. In that case I copy/share the relevant highlights to inbox
  • Watch/Read/Software/Golang: These four files are buckets for collecting links.
  • Shopping list.
  • TODO List.
  • Projects. Currently active project gets a separate file. All others are kept in this file. Whenever I come across an idea or link or tutorial that will help with anyone of the projects in back-burner, I add them here under their outline. It also acts as a record of my brainstorms, reasoning behind my decisions relating to that project, etc.
  • Current Project

I am neither very diligent nor an outrageous slack when it comes to processing these files. However from time to time there comes a situation where one of these files become larger than my liking. In such situations I move them to a different folder after attaching current date to its name.
